# Project Description
- 이 프로젝트 폴더는 머신러닝 / 계산화학 관련 연구 및 실험을 진행합니다
- 해당 프로젝트는 Universal Machine Learning Interatomic Potential(uMLIP)가 예측에 실패하는 High Entropy Alloy 시스템에서의 에너지 예측 불확실성을 Physical inspired representation을 과의 representatation alignment 를 측정하는 방식으로 간접적으로 추정하는 방법에 대해서 연구합니다. 더 나아가, 이 가설이 옳다면, aligment를 예측 불확실성을 대리하여 active learning 의 데이터 선택 기존으로 삼았을 때, 학습 효율의 높아지는 정도 또한 연구하고자 합니다.

# 일반적 workflow
- 다음의 단계를 통해서 representatation alignment가 uMLIP의 예측의 불확실성을 효과적으로 추정할 수 있는지 검증합니다
- (1) HEA 시스템으로 이루어진 원자 데이터셋 준비
- (2) uMLIP으로 에너지 예측을 진행 및 오차를 측정 / 저장
- (3) Physic-inspired representation 으로 해당 HEA 시스템을 각각 변환 및 저장. 
- (4) uMLIP 의 latent-vector를 추출하여 HEA 시스템을 각각 변환 및 저장.
- (5) 동일 시스템의 Physic-inspired representation 과 uMLIP 의 latent-vector 사이의 유사도를 측정 및 저장
- (6) 측정된 유사도와 uMLIP의 예측 오차 사이의 관계성을 측정

# 연구를 통해 검증되어야 하는 사항
- (1) 에너지 예측 오차와 가장 관계성이 높은 physical inspired representation은 무엇인가?
  - SOAP, Coulomb Matrix, Sine Matrix, Ewald Sum Matrix, MBTR 중 비교
- (2) 에너지 예측 오차와 (physical inspired representation - MLIP latent vector alignment) 사이의 유의미한 관계성이 존재하는가?
  - 5개 MLIP families (MACE, SevenNet, Orb, Equiformer, UMA) × 5개 physics-inspired representations 조합 분석
- (3) physical inspired representation - MLIP latent vector alignment 을 active learning에서 uncertainty proxy으로 사용하였을 때, 유의미한 학습 효율성 증가가 존재하는가?
- (4) MLIP family별 특성에 따라 가장 효과적인 physics-inspired representation이 다른가?

## 🆕 최근 업데이트 (2025.02.26)

### Sine Matrix 추출 이슈 해결

#### 문제 상황
dscribe의 `SineMatrix` descriptor가 ASE Atoms 객체와 호환성 문제 발생:
- **에러**: `AttributeError: 'Atoms' object has no attribute '_get_constraints'`
- **원인**: dscribe의 `System.from_atoms()` 메서드가 ASE Atoms의 private 메서드 `_get_constraints()`를 호출하려 하지만, ASE 3.27.0 버전에는 이 메서드가 존재하지 않음

#### 문제 분석
1. dscribe의 `/home/pn50212/anaconda3/envs/fairchem/lib/python3.10/site-packages/dscribe/core/system.py:89`에서 발생
2. `System.from_atoms()` 메서드가 ASE Atoms의 모든 속성을 복사하려 시도:
   ```python
   constraint=atoms._get_constraints()  # 문제 발생 지점
   ```
3. 실제로 Sine Matrix 계산에는 constraint 정보가 불필요:
   - 필요한 정보: symbols, positions, cell, pbc
   - 불필요한 정보: constraints, momenta, calculator 등

#### 해결 방법
dscribe의 `System.from_atoms` 메서드를 **monkey patch**하여 필수 정보만 복사하도록 수정:

```python
# src/representations/physics_inspired/sine_matrix.py의 setup() 메서드에서
from dscribe.core.system import System

def patched_from_atoms(atoms):
    """Patched version that doesn't call _get_constraints"""
    system = System(
        symbols=atoms.get_chemical_symbols(),
        positions=atoms.get_positions(),
        cell=atoms.get_cell(),
        pbc=atoms.get_pbc()
    )
    return system

# Monkey patch 적용
System.from_atoms = staticmethod(patched_from_atoms)
```

#### 결과
- ✅ dscribe의 SineMatrix를 그대로 사용하면서 ASE 호환성 문제 해결
- ✅ 10/10 테스트 구조 성공적으로 추출
- ✅ O adsorbate: 3,084/3,084 구조 성공적으로 추출 완료
- ✅ OH adsorbate: 4,084/4,084 구조 성공적으로 추출 완료
- ✅ 저장 위치: `datasets/25Cao/representations/sine_matrix/`

### MLIP별 Energy Head 분석 상황 (2025.02.27 업데이트)

#### ⚠️ SevenNet
- **Energy head layer**: `reduce_input_to_hidden` (출력이 latent vector)
- **Shape**: `(n_atoms, hidden_dim)` - per-atom features
- **문제점**: TorchScript 컴파일로 인한 hook 등록 불가
  - `RuntimeError: register_forward_hook is not supported on ScriptModules`
  - Monkey patching 시도했으나 실패
- **해결 필요**: TorchScript 우회 방법 또는 SevenNet 소스 수정 필요
- **mlp_infos.yml 업데이트**: ✅ 완료
- **상태**: 보류 (TorchScript 문제 해결 필요)

#### ✅ MACE
- **Energy head layer**: `readouts.1.linear_1` (출력이 latent vector)
- **Shape**: `(n_atoms, 112)` - per-atom features
- **Architecture**: Products → readouts.1.linear_1 (latent) → readouts.1.linear_2 (energy)
- **Hook 가능**: ✅ Yes (ScriptModule 문제 없음)
- **mlp_infos.yml 업데이트**: ✅ 완료
- **환경**: `conda activate mace`

#### ✅ Orb
- **Energy head layer**: `model._decoder` (입력이 latent vector)
- **Shape**: `(n_atoms, 256)` - per-atom features
- **Architecture**: GNN stacks → model._decoder (입력) → heads.energy.mlp → energy
- **Hook 가능**: ✅ Yes (decoder 입력 캡처)
- **mlp_infos.yml 업데이트**: ✅ 완료
- **환경**: `conda activate orb`

#### ⏳ Equiformer (HuggingFace 액세스 승인 대기중)
- **모델 수**: 3개 (31M, 153M, 121M)
- **예상 layer**: `norm_output` 또는 energy block
- **상태**: HuggingFace 저자 액세스 승인 대기 (2025.02.27 요청)
- **참고**: 모델 액세스 권한이 필요하여 승인 전까지 분석 불가

#### ✅ UMA (2025.02.27 완료)
- **모델 수**: 2개 (uma-s-1p1, uma-m-1p1)
- **Energy head layer**: `output_heads.energyandforcehead.head.energy_block.2` (출력이 latent vector)
- **Shape**: `(n_atoms, 128)` - per-atom features
- **Architecture**: backbone → energy_block.0 → energy_block.2 (latent) → energy_block.4 (energy)
- **Hook 가능**: ✅ Yes (일반 PyTorch 모듈)
- **mlp_infos.yml 업데이트**: ✅ 완료
- **환경**: `conda activate fairchem`
- **특징**: MLIPInferenceCheckpoint 형식, eSCNMDMoeBackbone 사용

### SevenNet TorchScript 문제 상세

#### 문제 요약
- SevenNet이 핵심 연산을 TorchScript로 컴파일 (`_compiled_main: RecursiveScriptModule`)
